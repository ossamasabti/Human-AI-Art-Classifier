# 🖼️ Human Art vs AI Art Classifier

[![Python 3.9+](https://img.shields.io/badge/python-3.9+-blue.svg)](https://www.python.org/downloads/)
[![PyTorch](https://img.shields.io/badge/PyTorch-2.0+-ee4c2c.svg)](https://pytorch.org/)
[![License: MIT](https://img.shields.io/badge/License-MIT-yellow.svg)](https://opensource.org/licenses/MIT)

This project aims to build a binary image classifier to distinguish between artworks created by humans and images generated by artificial intelligence (AI). Using a Vision Transformer (ViT) model fine-tuned on a custom dataset, the system can identify the origin of an image with high accuracy.

**The goal was to achieve a minimum accuracy of 70% on a validation set, which was successfully surpassed with a final accuracy of 84%.**

## 🚀 Features

*   **Robust Classification:** Distinguishes between human art and AI-generated images.
*   **State-of-the-Art Model:** Utilizes a pre-trained Vision Transformer (ViT-B/16) from `torchvision`.
*   **Efficient Fine-Tuning:** Employs transfer learning by training only the classification head, leading to fast convergence.
*   **Comprehensive Pipeline:** Includes data loading, exploratory data analysis (EDA), training, and evaluation scripts.
*   **Error Handling:** Gracefully handles corrupted or non-standard images during data loading.

## 📊 Performance

The model achieves solid performance on the validation dataset, demonstrating its ability to generalize.

*   **Validation Accuracy:** **84.0%**
*   **Training Accuracy:** 96.0%
*   **Objective:** Surpassed the initial target of 70%.

*Note: The gap between training and validation accuracy indicates some overfitting, which is expected given the limited dataset size.*

## 🧠 Methodology

1.  **Data Loading & Preprocessing:**
    *   Images are loaded from structured directories (`RealArt` for human art, `AiArtData` for AI art).
    *   All images are resized to **224x224** pixels, the standard input size for ViT.
    *   Data augmentation (random horizontal flip) is applied to improve generalization.

2.  **Model:**
    *   The core model is a `torchvision.models.vit_b_16`, pre-trained on the massive ImageNet dataset.
    *   **Transfer Learning:** The final classification layer is replaced with a new, randomly initialized linear layer for binary classification. Only this new layer is trained, leveraging the powerful features learned from ImageNet.

3.  **Training:**
    *   **Optimizer:** Adam (`lr=1e-4`)
    *   **Loss Function:** Cross-Entropy Loss
    *   **Batch Size:** 32
    *   **Epochs:** 5
    *   The model converges very quickly, often after the first epoch.

## 🛠️ Installation & Usage

### Prerequisites
*   Python 3.9 or higher
*   pip

### Setup

1.  **Clone the repository**
    ```bash
    git clone https://github.com/YOUR_USERNAME/human-ai-art-classifier.git
    cd human-ai-art-classifier
    ```

2.  **Create a virtual environment (recommended)**
    ```bash
    python -m venv venv
    source venv/bin/activate  # On Windows use `venv\Scripts\activate`
    ```

3.  **Install dependencies**
    ```bash
    pip install -r requirements.txt
    ```

### Running the Code

The project is structured as a Jupyter notebook. You can run it in Google Colab or locally.

1.  **Prepare your dataset:** Organize your images as shown in the [Dataset](#-dataset) section. If using Google Colab, upload your dataset as a ZIP file.
2.  **Open the notebook:** Launch `human_vs_ai_art_classifier.ipynb`.
3.  **Execute the cells:** Run the notebook cells sequentially. The first cell handles dataset upload (for Colab) or you can modify the `config.DATASET_PATH` to point to your local dataset.
4.  **View Results:** The final cell will output the training/validation accuracy and loss plots.

## ⚠️ Limitations & Future Work

### Current Limitations
*   **Dataset Size:** The relatively small dataset limits the potential for more extensive fine-tuning and contributes to overfitting.
*   **Performance Ceiling:** The accuracy plateaus around 84%, likely due to the limited diversity of artistic styles in the training data.
*   **Input Sensitivity:** The current pipeline is somewhat sensitive to non-standard image formats (e.g., RGBA, WebP), although basic error handling is in place.

### Proposed Improvements
*   **Short-Term:**
    *   Expand and diversify the dataset with more examples from various artistic movements and AI generators.
    *   Implement stronger regularization techniques (e.g., weight decay, dropout) to mitigate overfitting.
*   **Medium-Term:**
    *   Fine-tune more layers of the ViT model on a larger, more comprehensive dataset.
    *   Explore domain-specific pre-training on artistic datasets like WikiArt.
    *   Integrate image metadata (e.g., file size, palette analysis) as additional features.
*   **Long-Term:**
    *   Develop a dedicated, open-source benchmark dataset for this task.
    *   Build a user-friendly web application or API for real-time classification.
